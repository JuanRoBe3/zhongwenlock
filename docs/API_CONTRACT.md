# API Contract - ZhongwenLock

## Purpose

This document defines the HTTP API used by the ZhongwenLock web/PWA app and backend.

The API acts as the contract between:

- the web/PWA application;
- AWS API Gateway;
- AWS Lambda;
- DynamoDB.

In the first MVP, the Custom GPT tutor does not call the API directly. Instead, the user copies the JSON generated by ChatGPT and imports it into the web app.

In a future version, a Custom GPT Action may call `POST /sessions` directly.

---

## Endpoints

## POST /sessions

Used to import a completed study session.

In the MVP, the user copies the JSON generated by ChatGPT and pastes it into the web app. The app then sends it to this endpoint.

### Request

The request body must follow the structure defined in:

```text
samples/session-event-example.json
```

The payload includes:

- session metadata;
- score;
- errors;
- generated flashcards;
- generated exercises;
- mini-tests;
- simulated financial penalty.

### Processing

The backend must:

- validate the JSON payload;
- store the study session;
- store detected errors;
- store generated flashcards;
- store generated exercises;
- store mini-tests;
- update mastery scores;
- create a simulated ledger entry.

### Success Response

```json
{
  "status": "ok",
  "message": "Study session processed successfully",
  "session_id": "session_2026_07_05_001"
}
```

### Error Response

```json
{
  "status": "error",
  "message": "Invalid study session payload"
}
```

## GET /daily-review

Used by the app to retrieve the recommended review for the current day.

### Purpose

The backend selects the most relevant material based on:

- pending flashcards;
- weak concepts;
- recent mistakes;
- severity;
- previous review history.

### Success Response

```json
{
  "date": "2026-07-05",
  "flashcards": [],
  "exercises": [],
  "mini_tests": []
}
```

---

## POST /review-sessions

Used when the user completes a review session inside the app.

Instead of sending each answer one by one, the app sends all answers together at the end of the review session.

### Example Request

```json
{
  "event_type": "review_session.completed",
  "user_id": "demo-user",
  "review_session_id": "review_2026_07_05_001",
  "answers": [
    {
      "item_id": "FLASHCARD#2026-07-05#001",
      "item_type": "flashcard",
      "concept": "了",
      "result": "wrong",
      "difficulty_feedback": "hard"
    },
    {
      "item_id": "TEST#2026-07-05#001",
      "item_type": "multiple_choice",
      "concept": "是",
      "result": "correct"
    }
  ]
}
```

### Processing

The backend must:

- store the review session result;
- store the user answers;
- update mastery scores;
- update next review dates;
- update study streak;
- update the simulated ledger if needed.

### Success Response

```json
{
  "status": "ok",
  "message": "Review session processed successfully"
}
```

## GET /dashboard

Used by the app to retrieve global progress metrics.

### Success Response

```json
{
  "current_hsk_level": "HSK2",
  "global_mastery_score": 67,
  "weak_concepts": ["了", "Tono 3 vs Tono 4"],
  "study_streak_days": 5,
  "ledger_balance_eur": 8.5
}
```

---

## GET /ledger

Used by the app to retrieve the simulated financial ledger.

The MVP does not move real money.

### Success Response

```json
{
  "balance_eur": 8.5,
  "status": "simulated",
  "entries": [
    {
      "date": "2026-07-05",
      "amount_eur": 1.5,
      "reason": "Errors in HSK2 study session"
    }
  ]
}
```
